# Python3.10 运行机制

入口定义在 Modules/main.c

# 0x00 初始化运行环境

通过 pymain_init 来初始化 Python 运行环境，其中很多细节就不展开了。

# 0x01 运行 python

完成初始化后， Py_RunMain 函数会调用 pymain_run_python 函数开始运行

pymain_run_python 会根据不同的运行方式，选择不同的函数来运行，由于是恶意脚本检测，主要关注脚本方式运行。

  • pymain_run_file->pymain_run_file->pymain_run_file_obj->_PyRun_AnyFileObject

_PyRun_AnyFileObject 定义在 pythonrun.c

先来看 _PyRun_SimpleFileObject 函数

编译文件或语句得到 AST ，并调用 run_mod 执行。

执行文件为 pyc 文件

可以看到执行 pyc 文件不用编译，直接从文件中读出 PyCodeobject对象 ，再调用 run_eval_code_obj 函数执行字节码。

run_mod 函数基于 AST 得到 PyCodeObject 对象，再通过 run_eval_code_obj 函数调用 PyEval_EvalCode 执行 PyCodeObject 对象中的字节码指令序列。

看完 _PyRun_SimpleFileObject 函数，再来看 _PyRun_InteractiveLoopObject 函数

可以看到该函数核心调用的函数同 _PyRun_SimpleFileObject , 重点就在于 PyEval_EvalCode 如何执行字节码了。

PyEval_EvalCode 定义在 ceval.c

PyThreadState tstate->interp->eval_frame 在初始化时被 PyInterpreterState_New 函数初始化为 _PyEval_EvalFrameDefault

到了 _PyEval_EvalFrameDefault 函数，就开始执行一条一条的字节码了，该函数通过一个 for(;;) 和一个巨大的 switch-case 结构来循环执行每一条字节码指令。
