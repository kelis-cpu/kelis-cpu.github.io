# 一。目录遍历

目录遍历，也称为文件路径遍历，允许攻击者读取运行应用程序的服务器上的任意文件，包括应用程序代码，数据，后端系统的凭据以及敏感的操作系统文件，甚至可以写入文件，允许修改应用程序数据或行为，并最终控制服务器。

# 二。读取任意文件

Windows 上 ../和..\ 都是有效的遍历序列，Unix 是 ../

# 三。利用目录遍历漏洞的常见障碍

# 1. 目录遍历序列被剥离或阻止

  • 直接使用绝对路径访问

  • 使用嵌套遍历序列， ....//或....\/

  • 在某些情况下，在 URL 路径或 multipart/form-data 请求的文件名参数中，Web 服务器可能会将传输的给应用程序之前剥离任何目录遍历序列，可以通过 URL 编码，甚至 URL 双编码绕过， %2e%2e%2f,%252e%252e%252f,..%c0%af,..%ef%bc%8f

  • 要求文件名必须以指定文件开头，可以 filename=/var/www/images/../../../etc/passwd

  • 要求文件必须以指定扩展名结尾，可以使用空字节阻止文件路径， filename=../../../etc/passwd%00.png

# 四。如何阻止目录遍历

  • 最有效的方法是完全避免用户提供的输入传递给文件系统 API，可以重写许多执行此操作的应用程序功能。
  • App 在处理用户输入之前对其进行验证，与允许值的白名单进行比较或者验证输入是否包含允许的内容 (比如纯字母数字)。
  • 验证之后，App 将输入附加到基本目录并使用平台文件系统 API 来规范化路径，还应验证路径是否符合预期的基本目录开头。
