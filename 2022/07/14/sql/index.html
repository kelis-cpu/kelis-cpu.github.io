c. 时间盲注

burp intruder 的四种模式：

1.sniper

一个 payload set，不管 position 有几个，每个 position 逐项替换，替换第一个 position 时，其他 position 都保持原值。

2.battering ram

一个 payload set，多个 position 同时替换成同一值。

3.pitchfork

每个 position 都有属于自己的一个 payload set，如果每个 payload set 的 payload 个数不一样，多出来的 payload 不会被发送。

4.clutster bomb

每个 position 都一个属于自己的一个 payload set，请求的个数是各个 payload set 的笛卡尔积。

d.OAST

当详细的错误信息被禁用、结果被限制或过滤、减少查询数目、sql 查询进程和 http 请求进程异步时，利用其他传输信道获取数据，最常用是利用 dns 协议。

# SQL 注入不同的地方：

大部分的 sql 注入都存在 select 查询的 where 子句中，但是原则上 sql 注入可以发生在查询中的任何位置，以及不同的查询类型中，常见的其他位置是：

  • 在 update 语句中，在更新值或 where 子句中
  • 在 insert 语句中，在插入的值内
  • 在 select 语句中，在表名或者列名内，及 order by 子句中

# 3. 二次注入

通过将输入放在数据库来完成，但在存储数据的位置不会出现漏洞，当处理不同的 http 请求时，应用程序将会检索存储的数据并以不安全的方式将其合并到 sql 查询中。

# 数据库特定因素

sql 注入常用备忘录

数据库之间存在差异，检测和利用 sql 注入的技术在不同平台上的工作方式不同

  • 字符串连接的语法
  • 注释符
  • 堆叠查询
  • 特定于数据库的 API
  • 错误消息

# 如何防御 sql 注入

查询中的参数使用硬编码 + 参数化查询 (预编译语句)

注：硬编码（英语：Hard Code 或 Hard Coding）是指在软件实现上，将输出或输入的相关参数（例如：路径、输出的形式或格式）直接以常量的方式撰写在原始码中，而非在执行期间由外界指定的设置、资源、资料或格式做出适当回应。
