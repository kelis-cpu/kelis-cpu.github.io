# 一。信息泄露

  • 通过 robots.txt 或目录列表先是隐藏目录的名称、结构和内容
  • 通过临时备份提供对源代码文件的访问
  • 在错误消息中明确提及数据库表或列名
  • 不必要暴露高度敏感的信息，如信用卡详细信息
  • 在源代码中硬编码 API 密钥、IP 地址、数据库凭证
  • 通过 APP 行为的细微差异来暗示资源、用户名等是否存在

造成信息泄露的常见成因：

  1. 未能从公共内容中删除内部内容，如，标记中的开发人员评论时有时对生产环境中的用户可见。
  2. 网站不安全配置，如，未能禁用调试和诊断功能可以为攻击者获取敏感信息，默认配置也会使网站受到攻击，显示过于冗长的错误消息。
  3. 应用程序的设计和行为有缺陷，如，网站在发生不同的错误状态时返回不同的响应。

# 二。漏洞利用

# 1. 信息泄露测试

# 1.Fuzzing

发现可疑的参数，可以尝试提交意外的数据类型和特制的字符串，使用 Intruder 来自动执行，通过比较 HTTP 状态码，响应时间，长度区别响应，使用 grep 快速匹配关键字的出现。

# 2.scanner

burp scanner 可以在浏览网站时进行实时扫描，也可以自动对目标站点进行爬取和审计。

# 3.engagement tools

可以在任意 proxy，http 消息或 sitemap 中右键使用该工具

discover content

可以使用此工具来查找没在 sitemap 中自动出现的其他目录和文件。

# 4. 信息响应

利用错误消息提取可利用的数据，一般可以使用无效的参数值导致堆栈跟踪或包含有用消息的调试响应。

# 2. 信息泄露的常见来源

  • robots.txt/sitemap.xml
  • 网站目录列表

泄露敏感资源的存在和位置，如临时文件和故障转储。

  • 开发者注释

开发过程中，有时会将内嵌 HTML 注释添加都标记中，可能会暗示隐藏目录的存在或提供有关应用程序逻辑的线索。

  • 错误消息

错误消息可以揭示有关给定参数预期的输入或数据类型的信息，还可以提供网站使用的不同技术信息，网站正在使用的模板引擎、开源框架、数据库类型、服务器以及版本号。

  • 调试信息

调试信息包含开发攻击的重要信息：关键会话变量的值、后端组件的主机名和凭据、服务器的文件和目录名称、用于加密客户端传输的数据的密钥。调试信息可能会记录在单独的文件中。

  • 用户账户页面

某些网站可能允许攻击者利用这些页面来查看其他用户数据的逻辑缺陷，如根据用户参数确定要加载哪个用户的账户页面 /user/personal-info?user=carlos , 大多数网站都会采取措施防止攻击者更改此参数以访问任意用户的账户页面，但是加载单个数据项的逻辑不一定健壮。

  • 备份文件泄露源码

编辑原始文件时会生成临时备份文件，这些临时文件通过将～附加到文件名或添加不同的文件扩展名 (bak,swp)。

  • 配置不安全

第三方技术的广泛使用，大量的配置选项不一定被开发者理解；开发人员可能会忘记在生成环境中禁用各种调试选项，例如 HTTP TRACE 方法，如果启用，Web 服务器将通过在响应中回显收到的确切请求来响应使用 TRACE 方法的请求，内部身份验证标头的名称可能被反向代理附加到请求上。

  • 版本控制历史

几乎所有网站都使用某种形式的版本控制系统开发的，例如 Git，默认情况下，Git 醒目将其所有版本控制数据存储在一个名为.git 的文件夹中，有时，网站在生成环境中会公开此目录，可以利用 GitHack 恢复源码。 weget -r xxx/.git 递归下载.git
